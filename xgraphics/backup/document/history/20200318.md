JAVASCRIPT REFACTORING
======================

자바스크립트 라이브러리를 잠시 리팩터링 하려 합니다.
로딩을 구현하고 나니 로딩 후에 그래픽스 객체를 변경해 디스플레이를 해야 하는데,
나이스한 모습이 안되더군요.
로딩은 무거은 리소스를 외부에서 가지고 와서 그것을 작업할 수 있을 때까지
화면에 어떤 프로그램이 진행중임을 나타내는 표시를 해주는 것입니다.

아직도 정확하게 로딩은 구현되어 있지 않습니다.
중간에 건너 뛰는 것도 있구요.
중간에 건너 뛰는 것은 자연스럽게 보이도록 하기 위해서
약간의 트릭을 사용하였습니다.
외부의 프로그세스 까지.

 자바스크립트 리팩터링을 완료하였습니다.
 C++ 과 유사한 구조를 가지기 위해서 레퍼런스를 렌더링하도록 하였고,
 조인트에서 레퍼런스를 빌드하도록 하였습니다.
 그리고 홈월드 만의 특이한 좌표 계산 때문에, (다이렉트와 오픈지엘 간의 상호 호환 문제)
 렌더링을 상속하여 구현할 수 있도록 하였습니다.

신기하게도 코름에서 돌던 웹 워커는 파이어폭스에서는 멈추는 경향이 있네요.
웹 워커라 빠르게 소멸되던지 혹은 동작하지 않는 것 같습니다.

파이어 폭스 브라우져를 좋아하는데, 모바일 버전에서 조금 많이 느린 경향이 있네요.

HOMEWORLD TEXTURE
=================

홈월드2 텍스쳐를 분석하다 보니 매터리얼과 텍스쳐 간의 관계와
부거적인 쉐이더 관련된 로직을 정리할 필요가 있습니다.
그리고 홈월드의 기본 프리미티브 타입에서는 쉐이더의 인덱스를 가지고 있다가
오브젝트를 로딩할 때, 쉐이더 프로그램을 연결시켜야 할 듯 보입니다.

그리고 텍스쳐 관련된 것은 내부에서 텍스쳐를 여러개 가지고 있군요.
멀티플한 텍스쳐를 사용할 수 있도록 코드를 정리해야 할 것으로 보이구요.
여튼 더 분석이 필요합니다.
