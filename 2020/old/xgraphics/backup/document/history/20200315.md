EVENT LOOP & CUSTOM CONSOLE I/O
===============================

이벤트 프로세스를 작성하면서 무브와 카피의 생성자와 무브와 카피의 할당 연산자에 대해서 큐를 복제하지 않기로 하였습니다.
다른 이유는 없고, 이벤트는 전역적으로 하나인데, 그것 조차 복사되면 이벤트가 두번 호출되기 때문입니다.
하지만 고민스럽기는 합니다. 복제와 같은 것은 일관성이 있어야 하는데, 특수성을 가지는 클래스가 되니까

이벤트 생성 클래스를 짜고 있습니다.
포인터로 짜고 있는데, 역시 이 부분은 참조로 짜는 것이 좋겠습니다.

대략 이벤트 엔진의 구성은

1. 이벤트
2. 이벤트 객체
3. 이벤트 생성 객체
4. 이벤트 처리 객체
5. 이벤트 엔진

으로 구성될 것 입니다.
이벤트 엔진은 RUN 이란 메서드를 구현해야겠네요.

---

간단하게 이벤트 엔진의 골격을 만들었습니다.
우선 파일 입출력과 관련된 이벤트 생성 클래스를 만들고 생성 클래스를 엔진에 삽입하는 로직을 짜려고 합니다.

하루 종일 이벤트 시스템의 구현과 씨름을 했습니다.
우선은 윈도우즈의 구현을 빼려고 합니다.
차후에 멀티 플랫폼을 고려할 때, 조금 튜닝을 해야할 듯 보입니다.

이전에 콘솔 입출력이 어떻게 이루어지는지 궁금한 적이 있었습니다.
그저 사용만 하니까 어찌 돌아가는 지 모르고 `cin >> a;` 의 화살표가 가지는 의미를 잘 몰랐지요.
이번에 커스터마이징을 하면서 "실제 `cin` 의 구현이 이렇게 이루어졌겠구나!" 라고 생각해 보게 되었습니다.
cin 에서 멈추는 것이 많이 마음에 들지 않았는데,
이번에 콘솔입출력을 비동기로 구현하면서 이제 좀 흡족해 합니다.
내부는 하루 만에 구현하자고 결심해서 버그가 있을 것 입니다.
잘짜여진 이벤트 루프를 만드는 일은 사실 정말 그것만 잘만들려고 해도,
한달이 넘게 걸릴 것 입니다.
다만, 저도 좋은 이벤트 루프를 만들어 보려고 하고 있습니다.
시작일 뿐입니다.
TCP 서버까지 고려되면, 좋은 이벤트 루프를 지원하는 라이브러리가 나오겠지요!
